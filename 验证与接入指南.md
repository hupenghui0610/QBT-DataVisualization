# 是否需要验证当前项目？新项目如何接入？

本文与 [`多域名共用账号-部署说明.md`](多域名共用账号-部署说明.md)、[`账号体系需求规格-复刻用.md`](账号体系需求规格-复刻用.md) 配套，便于自检与接新站。

---

## 一、当前项目要不要验证？

**建议做**，尤其是改过 **CORS / `ALLOWED_ORIGINS`** 或准备上生产时。不必全量回归，以**冒烟**为主。

| 场景 | 建议验证 |
|------|----------|
| **未配置** `ALLOWED_ORIGINS` | 与以前一致：本站 [`index.html`](index.html) 登录、`ping`、拉数接口正常即可。 |
| **已配置** `ALLOWED_ORIGINS` | 用**白名单内的**浏览器 Origin 访问：登录、`POST /api/auth/ping`、任一需登录的 `GET /api/data/*`；再故意用**未列入**的 Origin 跨域请求，应被 CORS 拦住（与 [`functions/_lib/http.js`](functions/_lib/http.js) 中 `resolveCorsOrigin` 行为一致）。 |
| **仅本仓库部署**、无多站跨域 | 若生产不配白名单，验证成本可降到最低：走一遍登录与主数据页。 |

本地可用 `wrangler pages dev`（若你平时用）或直连已部署域名验证；关键看**浏览器控制台无 CORS 报错**、**401/业务 JSON 能正常解析**。

### 冒烟清单（可选，配置白名单后建议勾）

1. Cloudflare 上为 API 项目设置 `ALLOWED_ORIGINS`，包含本站与待接业务站完整 Origin。
2. 浏览器打开**白名单内**站点：登录 → 主流程拉数正常。
3. 用未列入 Origin 的测试页或 `curl`/另一域名仅验证：浏览器跨域请求应无法读响应体（CORS 失败属预期）。

## 二、新项目怎么接入（推荐方案 A：单一 API 入口）

目标：**用户库与 JWT 只维护在现有这套 Pages + Functions + D1 上**，新业务只是另一个前端站点。

1. **API 根**  
   新业务前端请求统一指向现有部署，例如 `https://qbt-datavisualization.pages.dev`（以你实际域名为准）。实现方式二选一或组合：  
   - 构建时写死与 [`js/auth-client.js`](js/auth-client.js) 同样的 `getApiBase()` 逻辑；或  
   - 在新项目里复制/引用同一套：**`REMOTE_API_ORIGIN` + `localStorage['QBT_API_ORIGIN']` 覆盖**（见该文件注释）。

2. **鉴权**  
   登录仍走 `POST /api/auth/login`，Token 存 **`localStorage` 键 `xbs_token`**（或新项目若改名，则所有 `Authorization: Bearer` 与存键一致即可）。后续请求头：`Authorization: Bearer <token>`，`Content-Type: application/json`（与现网一致）。

3. **CORS（必做）**  
   在**承载 API 的 Cloudflare Pages 项目**环境变量中增加新业务域名，例如：  
   `ALLOWED_ORIGINS=https://现有站,https://新业务站`  
   （完整 Origin，含 `https` 与端口。）详见 [`多域名共用账号-部署说明.md`](多域名共用账号-部署说明.md)、[`DEPLOY-CLOUDFLARE.md`](DEPLOY-CLOUDFLARE.md) §4.1。

4. **不要**在新项目再绑一套用户 D1、再配另一个 `JWT_SECRET`（除非你走「方案 B：每站一套 Functions 但同一 D1 + 同一密钥」）。

5. **体验说明**  
   各站 `localStorage` 独立，用户在新站**一般要再登录一次**；若要「只登一次多站通用」，需另做 SSO（本仓库未实现 code 换 token，见 [`多域名共用账号-部署说明.md`](多域名共用账号-部署说明.md)）。

### 新项目接入清单

- [ ] 前端 API 基址指向现有 Pages API 域名（与 `auth-client` 行为一致）。
- [ ] Cloudflare 上 `ALLOWED_ORIGINS` 已包含新业务站完整 Origin，并已重新部署 Functions。
- [ ] 新站走一遍登录与至少一个需鉴权接口，控制台无 CORS 错误。
