# 上线检查清单（QBT API + 新项目独立域名）

与 [QBT账号对接环境说明.md](QBT账号对接环境说明.md)、[多域名共用账号-部署说明.md](多域名共用账号-部署说明.md) 配套。

## 1. 部署 QBT Pages（含 Functions）

在本仓库根目录任选一种方式提供 **`JWT_SECRET`**（与线上一致）后执行：

**方式 A（推荐，一次配置）**：复制 [`.env.deploy.local.example`](.env.deploy.local.example) 为 **`.env.deploy.local`**，填写一行 `JWT_SECRET=你的密钥`（该文件已 gitignore，勿提交）。

```powershell
npm run deploy:prod
```

测试环境部署 **`npm run deploy:test`** 与上共用同一 `.env.deploy.local`、同一 **`JWT_SECRET`**（本仓库约定；D1 仍为测试库）。

**方式 B**：每次在 PowerShell 设置环境变量：

```powershell
$env:JWT_SECRET="你的与线上一致的密钥"
npm run deploy:prod
```

说明见 [scripts/deploy-pages-prod.js](scripts/deploy-pages-prod.js)。若控制台已存在同名 `JWT_SECRET` Secret 导致冲突，按脚本注释删除控制台中的 Secret 后重试。

## 2. 新项目正式域名加入 CORS 白名单

当前生产配置在 [wrangler.toml](wrangler.toml) 的 **`ALLOWED_ORIGINS`**。新项目上线获得 **https 域名** 后：

1. 在 `ALLOWED_ORIGINS` **末尾追加**（逗号分隔，无空格）：`,https://新项目域名`  
2. 重新执行 `npm run deploy:prod`（同上需 `JWT_SECRET`）。

若从 **局域网 IP**（如 `http://192.168.1.5:5500`）本地调试连生产 API，也需把该 Origin 追加进 `ALLOWED_ORIGINS`，否则会 CORS 失败。

## 3. 新项目侧

- API 根：`https://qbt-datavisualization.pages.dev`  
- Token：`localStorage` 键 `xbs_token`；`Authorization: Bearer`  
- `POST /api/auth/login`、`POST /api/auth/ping`

## 4. 验收

浏览器打开新项目域名 → 登录 → 控制台无 CORS 错误 → 业务接口正常。
