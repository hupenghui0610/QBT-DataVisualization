# QBT数据可视化后台接口描述

## 项目概述

本项目基于 **Cloudflare Pages Functions** 构建，提供认证、数据查询、管理员功能等API接口。数据主要来源于飞书开放平台电子表格，通过服务端代理读取并处理后返回给前端。

---

## 架构说明

### 技术栈
- **运行环境**: Cloudflare Pages Functions (Edge Runtime)
- **数据存储**: Cloudflare D1 (SQLite)
- **外部数据源**: 飞书开放平台电子表格
- **认证方式**: JWT Token

### 目录结构
```
functions/
├── api/
│   ├── auth/          # 认证相关接口
│   ├── admin/         # 管理员接口
│   └── data/          # 数据查询接口
└── _lib/              # 公共库
    ├── crypto.js      # 加密/JWT
    ├── feishu.js      # 飞书API封装
    ├── http.js        # HTTP工具
    ├── session.js     # 会话管理
    └── login-security.js # 登录安全
```

## 一、认证接口 (Auth)

### 1.1 用户登录
- **路径**: `/api/auth/login`
- **方法**: POST
- **功能**: 用户登录验证，返回JWT Token

**请求参数 (Body)**:
```json
{
  "phone": "13800138000",
  "password": "用户密码"
}
```

**响应数据**:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": 1,
    "name": "张三",
    "phone": "13800138000",
    "is_admin": false
  }
}
```

**业务逻辑**:
1. 验证手机号和密码格式
2. 查询当前IP和账号的登录限制状态
3. 检查是否被临时锁定（IP或账号级别）
4. 验证密码（bcrypt比对）
5. 生成JWT Token（有效期24小时）
6. 记录登录成功/失败事件到安全日志

---

### 1.2 修改密码
- **路径**: `/api/auth/change-password`
- **方法**: POST
- **功能**: 修改当前用户密码

**请求参数 (Body)**:
```json
{
  "oldPassword": "原密码",
  "newPassword": "新密码"
}
```

**响应数据**:
```json
{
  "success": true
}
```

**业务逻辑**:
1. 验证JWT Token获取当前用户
2. 验证原密码正确性
3. 使用bcrypt加密新密码
4. 更新数据库用户密码

## 三、业务数据接口 (Data)

### 3.1 行业大盘数据
- **路径**: `/api/data/features-output`
- **方法**: GET
- **功能**: 获取行业大盘多维度分析数据

**数据源**: 飞书电子表格(Sheet1)
- 表格结构：渠道、价格段、日期、销量、销售额

**响应数据**:
```json
{
  "数据说明": {
    "数据源": "feishu:sheet_id!A1:E20000",
    "提取时间": "2026-04-09T10:00:00.000Z",
    "原始行数": 5000
  },
  "维度一_渠道维度": {
    "说明": "各渠道销量合计、销售额合计、占比、客单价（已剔除0-1k）",
    "全盘销量合计": 100000,
    "全盘销售额合计": 500000000,
    "按渠道": [
      {
        "渠道": "京东",
        "销量": 40000,
        "销售额": 200000000,
        "销量占比": 0.4,
        "销售额占比": 0.4,
        "客单价": 5000
      }
    ]
  },
  "维度二_价格段维度": {...},
  "维度三_时间趋势": {...},
  ...
}
```

**计算逻辑**:
1. **数据清洗**: 从飞书读取原始数据，解析Excel日期格式
2. **过滤**: 剔除0-1k价格段数据（低价值商品）
3. **维度一（渠道维度）**: 按渠道分组聚合，计算各渠道销量、销售额、占比、客单价
4. **维度二（价格段维度）**: 按价格段分组，统计各价格段结构
5. **维度三（时间趋势）**: 按月聚合，计算环比、同比增长率
   - 环比 = (本月 - 上月) / 上月
   - 同比 = (本月 - 去年同期) / 去年同期
6. **维度四-七**: 渠道×价格段×时间的交叉分析
7. **维度八（客单价衍生）**: 各维度客单价对比
8. **维度九（增长与占比）**: 占比环比变化
9. **维度十（集中度与排名）**: CR1/CR2/CR3集中度指标

- **方法**: GET
- **功能**: 获取前十大品牌分析数据

**数据源**: 飞书电子表格(Sheet2)
- 表格结构：渠道、品牌、价格段、日期、销量、销售额

**响应数据**:
```json
{
  "数据说明": {
    "数据源": "feishu:sheet_id!A1:G20000",
    "提取时间": "2026-04-09T10:00:00.000Z",
    "原始行数": 8000,
    "前十大品牌": ["学而思", "科大讯飞", "作业帮", "小猿", "小度", "BBK/步步高", "SEEWO/希沃", "BOE/京东方", "清北道远", "智能精准学"]
  },
  "品牌销量/销额": {
    "按品牌": [
      {
        "品牌": "学而思",
        "销量": 15000,
        "销售额": 75000000
      }
    ]
  },
  "品牌渠道占比": {...},
  "品牌市占率": {...},
  "品牌客单价": {...},
  "品牌价格段分布": {...},
  "品牌与日期": {...},
  "品牌与渠道与日期": {...}
}
```

**计算逻辑**:
1. **品牌名称归一化**: 处理品牌别名（如"讯飞"→"科大讯飞"）
2. **固定品牌名单**: 只分析预定义的十大品牌
3. **品牌销量/销额**: 按品牌汇总销量和销售额
4. **品牌渠道占比**: 各品牌在各渠道的销量占比
5. **品牌市占率**: 各品牌市场占有率 = 品牌销量 / 行业总销量
6. **品牌客单价**: 品牌销售额 / 品牌销量
7. **品牌价格段分布**: 各品牌在不同价格段的销量分布
8. **时间序列**: 按月统计品牌销量趋势

## 四、飞书数据读取机制

### 4.1 认证流程
1. 使用 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 获取 tenant_access_token
2. Token缓存2小时，提前120秒刷新
3. 存储在内存变量中，不持久化

### 4.2 数据读取
- **API版本**: 飞书 Sheets API V2
- **读取方式**: GET /open-apis/sheets/v2/spreadsheets/{token}/values/{range}
- **参数**: `valueRenderOption: 'UnformattedValue'`（获取公式计算后的值）
- **范围格式**: `sheet_id!A1:ZZ20000`

### 4.3 数据处理流程
```
飞书API → 原始数据 → 数据清洗 → 维度计算 → JSON响应
```

### 4.4 核心计算模块
**industry-data-builder.cjs**:
- `parseDaPanRowsFromValues()`: 解析大盘原始数据
- `parseBrandRowsFromValues()`: 解析品牌原始数据
- `dimension1_channel()` ~ `dimension10_ranking()`: 10维度分析函数
- `buildDaPanPayloadFromValues()`: 构建大盘响应
- `buildBrandPayloadFromValues()`: 构建品牌响应

## 八、错误码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证或Token过期 |
| 403 | 无权限（非管理员） |
| 429 | 请求过于频繁（限流） |
| 500 | 服务器内部错误 |
| 502 | 飞书API调用失败 |
| 503 | 服务器未配置 |
