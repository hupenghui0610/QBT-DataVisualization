# QBT数据可视化平台 - 技术说明文档

## 一、项目概述

**QBT数据可视化平台**是一个基于Web的行业数据分析系统，主要功能包括：
- 行业大盘数据展示（销量、销售额、渠道分布）
- 品牌销量/销额趋势分析
- 品牌市场占有率排行
- 多渠道数据整合（天猫、京东、抖音等）
- 日销、月销、同比等多维度对比

**技术栈：** HTML5 + Vanilla JS + ECharts + Cloudflare Pages/Workers/D1 + 飞书开放平台

## 三、前端技术实现

### 3.1 技术选型

| 技术 | 版本/说明 | 用途 |
|------|----------|------|
| HTML5 | Living Standard | 页面结构，语义化标签 |
| CSS3 | Flexbox + Grid | 响应式布局，自适应屏幕 |
| JavaScript | ES2018+ | 业务逻辑，数据处理，图表交互 |
| ECharts | 5.x | 数据可视化（折线图、柱状图、饼图、热力图） |
| FontAwesome | 4.7 | 图标库 |

### 3.2 架构特点

**纯静态前端，无构建工具**
- 不使用 React/Vue/Angular，减少依赖
- 原生 JavaScript 直接操作 DOM
- 代码结构：模块化函数组织，按图表维度拆分

**响应式设计策略**
```css
/* Mobile-first 媒体查询 */
@media (max-width: 768px) { /* 移动端适配 */ }
@media (min-width: 769px) and (max-width: 1024px) { /* 平板 */ }
@media (min-width: 1025px) { /* 桌面端 */ }
```

**数据流**
1. 页面加载：骨架屏 → 初始化 ECharts 实例
2. 数据获取：Fetch API → `/api/data/*` → JSON 响应
3. 数据处理：前端聚合、计算同比、格式化
4. 图表渲染：ECharts.setOption()

### 4.3 飞书数据集成

**数据获取流程**
```
用户请求 → 检查缓存 → 缓存命中？
    │
    ├─ 是 → 返回缓存数据
    └─ 否 → 调用飞书 API
              │
              ▼
        1. 获取 AccessToken
        2. 读取电子表格数据
        3. 数据清洗/转换
        4. 存入 D1 缓存
        5. 返回数据
```

**飞书 API 调用**
```javascript

// Range 示例：sheet1!A1:E20000
```

## 五、数据库设计

### 5.1 D1 数据库表结构

**users 表**
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,      -- 登录用户名
  phone TEXT,                     -- 手机号（可选）
  password_hash TEXT NOT NULL,    -- PBKDF2-SHA256 哈希
  token_version INTEGER DEFAULT 0,-- JWT 版本控制
  is_admin INTEGER DEFAULT 0,     -- 0=普通用户, 1=管理员
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**login_security 表**
```sql
CREATE TABLE login_security (
  scope_type TEXT NOT NULL,       -- 'ip'|'user'|'device'
  scope_key TEXT NOT NULL,        -- IP地址/用户名/设备ID
  consecutive_failures INTEGER DEFAULT 0,
  blocked_until DATETIME,         -- 封禁截止时间
  last_attempt_at DATETIME,
  PRIMARY KEY (scope_type, scope_key)
);
```

**data_cache 表**
```sql
CREATE TABLE data_cache (
  cache_key TEXT PRIMARY KEY,     -- 接口路径+参数哈希
  cache_data TEXT NOT NULL,       -- JSON 序列化数据
  expires_at DATETIME NOT NULL,   -- 过期时间
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 缓存策略

| 数据类型 | 缓存时间 | 更新策略 |
|---------|---------|---------|
| 行业大盘 | 48h | 首次请求生成，后台定时预热 |
| 品牌数据 | 48h | 同上 |
| 日销数据 | 2h | 实时性要求高，缩短TTL |
| GMV/GSV | 2h | 同上 |

## 六、账号体系与安全机制

### 6.1 JWT 认证流程

```
1. 登录请求
   POST /api/auth/login
   Body: { name, password }

2. 后端验证
   - 查询 users 表
   - PBKDF2 验证密码哈希
   - 检查 login_security 是否被封禁

3. 生成 Token
   JWT.sign({ userId, version, exp }, JWT_SECRET)

4. 返回客户端
   { token, user: { id, name, is_admin } }

5. 客户端存储
   - LocalStorage: token
   - Cookie: token (httpOnly=false, 用于JS读取)

6. 后续请求
   Header: Authorization: Bearer <token>
```

### 6.2 密码安全

**PBKDF2-SHA256 哈希**
```
格式: pbkdf2$sha256$100000$<salt>$<hash>

参数:
- 算法: SHA-256
- 迭代次数: 100,000
- Salt: 随机 32 字节
- Hash: 256 位

验证过程:
1. 拆分存储的哈希字符串
2. 提取 salt
3. 使用相同参数计算输入密码的哈希
4. 对比结果
```

### 6.3 登录安全控制

**防暴力破解机制**
```javascript
// IP 级别限制
同一 IP 连续失败 5 次 → 封禁 15 分钟

// 用户级别限制  
同一用户连续失败 5 次 → 封禁 15 分钟

// 封禁逻辑
if (failures >= 5) {
  blocked_until = now + 15分钟
}
```

### 6.4 其他安全措施

| 措施 | 实现方式 |
|------|---------|
| CORS | 白名单限制，只允许特定域名 |
| XSS | 输出转义，Content-Security-Policy |
| HTTPS | 强制 TLS 1.2+，HSTS 头部 |
| Secrets | 环境变量存储，不写入代码 |

### 7.2 数据隔离

**数据库完全隔离**
- 正式服账号无法登录测试服
- 测试服数据操作不影响正式服
- 测试服使用独立的数据库实例

**飞书数据源**
- 两环境共用同一套飞书表格（只读）
- 数据源由业务人员在飞书维护
- 系统只读取数据，不修改

## 九、性能优化

### 9.1 前端优化

- **代码分割**：按页面维度拆分，首页只加载必要资源
- **懒加载**：图表组件按需初始化
- **数据聚合**：服务端聚合后返回，减少前端计算
- **骨架屏**：加载时展示占位图，提升感知速度

### 9.2 后端优化

- **边缘缓存**：D1 数据库缓存飞书 API 响应
- **CDN 缓存**：静态资源全球分发
- **Workers 缓存**：重复请求复用计算结果
- **连接池**：复用飞书 API Token，减少鉴权请求

### 9.3 监控指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 首屏加载 | < 2s | Lighthouse |
| API 响应 | < 500ms | Cloudflare Analytics |
| 缓存命中率 | > 80% | 日志统计 |
